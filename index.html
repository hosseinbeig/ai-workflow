Phase 1

Strategic Planning

[ARCHITECT]

Persona: Lead System Architect (architect.mdc)

Input Jira Ticket ID + Project Context
Action Analyzes requirements, maps foundation
Output: docs/plans/[TICKET_ID].md containing:
  • Architecture Trees: Directory structure for the feature.

Phase 2

Design Fidelity

[ATOMIC_DESIGN_SYSTEM]

Persona: Design System Lead (atomic-design-system.mdc)

  • Input: Figma URL + Architect’s Plan.
  • Process: Performs a "Grep Audit" of codebase to prevent duplication.
  • Ruleset: Enforces Atomic Design (Atoms → Molecules → Organisms).
  • Output: UI Specification added to Plan Artifact, including exact Tailwind tokens and Props APIs.

The Critical Safety Gate (Human-in-the-Loop)

Zero-Risk Execution: The AI is physically barred from writing code until a human approves the Plan Artifact.

Phase 3

Engineered Implementation

[WEB]

Persona: Senior Frontend Engineer (web.mdc)

Input: The Approved Plan + Internal Skills Library
api-call.md Enforces Axios singletons and safeRequest patterns.
state-management.md Enforces TanStack Query for APIs and Zustand for UI state.

Output: Type-safe Next.js code that matches the plan 1:1.

Phase 4

Automated QA

[CODE_REVIEW]

Persona: Lead Code Reviewer (reviewer.mdc)

  • Input: Source Code + Original Plan
  • Process: "Audit Mode" scan. It does not change code; it identifies deviations.

> "Line 24 in UserCard.tsx violates the Atomic Design rule for Organisms."

> "Data contract in api/user.ts does not match the Architect's JSON spec."

Phase 5

Security Hardening

[SECURITY]

Persona: Security Auditor

Task: Final scan of the web/ directory focusing on:

  • XSS/CSRF Prevention: Validating input handling.
  • Server Action Security: Ensuring proper auth checks.
  • Secrets Check: Verifying no env vars leak to client.
